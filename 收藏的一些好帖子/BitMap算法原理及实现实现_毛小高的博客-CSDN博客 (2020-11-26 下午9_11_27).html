BitMap算法原理及实现实现

1. bitmap是什么

bitmap是一个十分有用的结构。所谓的Bit-map就是用一个bit位来标记某个元素对应的Value， 而Key即是该元素。由于采用了Bit为单位来存储数据，因此可以大大节省存储空间。

2. bitmap优势

举个例子，有一个无序有界int数组{1,2,5,7},初步估计占用内存44=16字节，这倒是没什么奇怪的；但是假如有10亿个这样的数呢，10亿4/(102410241024)=3.72G左右。如果这样的一个大的数据做查找和排序，那估计内存也崩溃了，有人说，这些数据可以不用一次性加载，那就是要存盘了，存盘必然消耗IO。

如果用BitMap思想来解决的话，就好很多。一个byte是占8个bit，如果每一个bit的值就是有或者没有，也就是二进制的0或者1，如果用bit的位置代表数组值有还是没有，那么0代表该数值没有出现过，1代表该数组值出现过。也可以描述数据。具体如下图：


现在假如10亿的数据所需的空间就是3.72G/32，一个占用32bit的数据现在只占用了1bit，节省了不少的空间，排序就更不用说了，一切显得那么顺利。这样的数据之间没有关联性，要是读取的，你可以用多线程的方式去读取。时间复杂度方面也是O(Max/n)，其中Max为byte[]数组的大小，n为线程大小。

3. bitmap算法代码实现

第一步： 构建特定长度的byte数组（new byte[capacity / 8 + 1]），其中 capacity为整数数组长度（如：10亿个数字、40亿个数字等）；

byte[] bits = new byte[getIndex(n) + 1];

第二步 ： 计算数字num在byte[]中的位置（num/8）； 

  1. 	/**
  2. 	 * num/8得到byte[]的index
  3. 	 * @param num
  4. 	 * @return
  5. 	 */
  6. 	public int getIndex(int num){
  7. 		return num >> 3;
  8. 	}
第三步： 计算数字num在byte[index]的位置（num%8）； 
  1. 	/**
  2. 	 * num%8得到在byte[index]的位置
  3. 	 * @param num
  4. 	 * @return
  5. 	 */
  6. 	public int getPosition(int num){
  7. 		return num & 0x07;
  8. 	}
第四步： 将所在的位置从0变成1.其他位置不变； 
  1. 	/**
  2. 	 * 标记指定数字（num）在bitmap中的值，标记其已经出现过<br/>
  3. 	 * 将1左移position后，那个位置自然就是1，然后和以前的数据做|，这样，那个位置就替换成1了
  4. 	 * @param bits
  5. 	 * @param num
  6. 	 */
  7. 	public void add(byte[] bits, int num){
  8. 		bits[getIndex(num)] |= 1 << getPosition(num);
  9. 	}


第五步： 判断指定数字num是否已存在； 
  1. 	/**
  2. 	 * 判断指定数字num是否存在<br/>
  3. 	 * 将1左移position后，那个位置自然就是1，然后和以前的数据做&，判断是否为0即可
  4. 	 * @param bits
  5. 	 * @param num
  6. 	 * @return
  7. 	 */
  8. 	public boolean contains(byte[] bits, int num){
  9. 		return (bits[getIndex(num)] & 1 << getPosition(num)) != 0;
  10. 	}


第六步：重置某一数字对应在bitmap中的值； 
  1. 	/**
  2. 	 * 重置某一数字对应在bitmap中的值<br/>
  3. 	 * 对1进行左移，然后取反，最后与byte[index]作与操作。
  4. 	 * @param bits
  5. 	 * @param num
  6. 	 */
  7. 	public void clear(byte[] bits, int num){
  8. 		bits[getIndex(num)] &= ~(1 << getPosition(num));
  9. 	}
